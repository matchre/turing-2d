Welcome on this emulator of a 2D Turing machine ! Let us learn together how this works on some concrete examples.

Concept

You may not be familiar with the Turing Machine concept.

Explanations

A reminder is therefore required. The machine has a tape composed of data cells on which a symbol is written. It also has an instruction set which describes a program. At each iteration, the machine performs actions. These are typically: change the status of the machine, move the date tape cursor, write data onto the tape. The next action is chosen according to the instruction set, the current machine state, or the symbol read on the date tape. From this simple mechanism the operations of a complex computer can be reproduced. In fact, all algorithms can be implemented.

Here we use a special machine, with a two dimensional tape. The cursor can move horizontally and vertically. Let us the term map to name such a band. A second map is used to encode the instruction set. It is also covered by a cursor. The cursor position determines the state of the machine. The program card and the data use the same set of symbol.

This symbol set if relatively simple. It comprises arrows coding the cursor displacement on the data map ( ), arrows coding the displacement on the instruction map ( ), while the default behavior is from left to write after reading a symbol, and finally color symbols, without any meaning as instruction beyond a default displacement ( ... ). Let us add two more complex symbols, y order to have a full functionality. The circle, ( ) is the ``conditional symbol´´ as a function of the symbol that follows. When the machine encounters this symbol, the next symbol is read, and if the symbol on the data map is the same as this next symbol, the next instruction symbol is the one on the left, otherwise it is the one below. Finnaly the cross ( ) allows to write on the data map a symbol, the symbol lefwards to the cross.

Sort

Yet another little game on integers. Let us sort an integer sequence.

Representation

Yet another time, the integer input list is put in column.

At the end, the goal is to have the integers sorted top-down from the biggest to the smallest.

Procedure

Prerequisites

The proposed solution is a kind of mixture betweem the so called insertion-sort and bubble-sort.

It is based on the following atomic operation: compare two consecutive integers and exchange their locations, if not correctly sorted. Try to design the operation on your own. This is in fact quite simple:

  • The current position is set at the end of the lower number.
  • If the upper number value is lower, it is incremented while the lower number is decremented ( we only have to test the cell above the end of the second number is empty )
  • We then shift one cell leftward and repeat the operation.

Let us take care about the case the integer is zero!

Loop

This being done, we now ahe to repeat this operation a certain amount of time in a clever way. Let us take a few seconds to think about it ..

As usual when programming, we are going to see how to segmentate the problem.

Let us assume that we have already obtained a partition of the list with n integers already sorted, and let us consider the n+1 integer below. We only have to do our favorite sorting operation between the n and n+1 integers, if they are well sorted nothing is to be done ( the n+1 integer sequence is well sorted ), otherwise the last element is going to "climb up" one stage. It is then compared to the n-1, integer and switched upward if needed, and so on. We are thus certain, that way, that the n+1 integer sequence is well sorted.

Great! And now? We repeat the procedure with the next element, until all element have travelled upwards ( as bubbles in a bear mug, yielding the "bulle sort" method name )

We still have to verify that the initial assumption is verified, i.e., that the n first elements are sorted. This is trivial if we start with n = 1 !

At the implementation level, we propose to mark the already sorted partition in order to make things readable. Green cells are replaced by yellow cells, when sorted.

Division

Since we have implemented the multiplication, the division will not be that complicated!

As for the multiplication, we are going to set the two integers A and B one above the other. The goal is to obtain the quotient as the 3rd integer and the remainder as the 4th one.

The expected result is thus 7 = 3 * 2 + 1 in this case

Procedure

We simply are going increment the quotient each time we can substract B to A. We also are going to mark the substracted units at each iteration on A.

Implementation
  • While A and B are not entirely marked, we marked a unit on each.
  • If B is entirely marqued, we increment the quotient, unmarked B, and repeat the previous step..
  • If A is entirely marked, then the quotient is obtained, and we report the remainder which corresponds to the present marks on the divider B.

We use towo colors blue and yellow, to represent a binary number. Yellow for 1. Blue for 0. The number reads from left to right, from the higher bit to the lower bit. No need of marker at the beginning here. Integers are coded on 6 bits, here, i.e., we are going to have 6 cells.

Notice that 63 is the highest number that can be represented by such a system

Procedure

The binary addition is relatively simple. In this case, we are going to implement the operation bit after bit.

When adding two bits, three cases are possible:

  • If the two bits are zero, the result is zero.
  • If one bit is not zero, and the other is zero, the result is one.
  • If both bits are not zero, the result is zero again and a carry is to be added on the next two bits addition
As a consequence, the situation is a bit more complex since the carry is to be taken into account at each stage.

The logical operation that gives the two bit addition result is the xor, the one that calculates the carry is the and.

Universal machine

Well! We have already done nice things, but with some extra energy, we could even reproduce the behavior of an Arithmetic Logic Unit, i.e., the processor component dedicated to logical operations. We even could simulate a whole computer!

You may find it goes too far, since after all it is your computer that makes such program runing, thus simulating the behavior of the computer itself.

In fact, we are not ready to stop! Let us create a Universal Turing machine, i.e., a Turing machine that simulates the behavior of . . a Turing machine!

In fact, the data map uses the same symbols as the instruction map, we thus can write the program instructions on the data map, and then design a universal program that interpret the data map instructions.


We are going to use a marker to locate the ongoing pseudo-instruction on the data map. We thus must leave an empty line below each instruction.

The main operation is to test the symbol related to the marker and implement the related operation.

If it is a displacement arrow, it is easy and we simply have to move the marker accordingly. Other instruction are more complex but not intractable.
